# Överblick
Vi går igenom gränssnitt i kontexten av en webbapplikation. Eftersom eleverna inte har fått en introduktion till HTML och CSS så börjar vi med att gå igenom grundläggande om hur HTML fungerar och skrivs, för att sedan visa hur javascript interagerar med Document Object Model (DOM).

# Syfte
Att ge eleverna en grund till att börja skapa sina egna gränssnitt och förstå hur Javascript kan användas utanför en konsolmiljö.

# Möjligheter för bedömning
*Vilka möjligheter har du att göra formativa bedömningar av elevernas färdigheter och kunskaper?*

Jag kommer kunna se elevernas progression under våra code-alongs, och sedan hur de löser samarbetsuppgifterna. Då kan jag också ge feedback till dem.

## Kursmål och betygskriterier

### Centralt innehåll:
- Programmeringens möjligheter och begränsningar utifrån datorns funktionssätt.
- Gränssnitt för interaktion mellan program och användare.

### Betygskriterier:
Dessutom väljer eleven **med viss säkerhet** ett uttryckssätt som är anpassat för att på ett **tillfredsställande** sätt interagera med den avsedda användaren.

# Mål
*Vad ska eleverna ha lärt sig innan lektionens slut? Alternativt vad ska vi ha åstadkommit?*

>[!success] Använda javascript tillsammans med HTML
Alla elever ska ha förståelse för hur de kan skapa HTML och javascript som förändrar, lägger till, och tar bort HTML-element på en existerande hemsida. De ska dessutom vara redo för nästa lektioner där vi utvidgar konceptet av att använda javascript i en webbapplikation.

>[!success] Javascript
Vi ska ha använt metoderna `document.getElementByID`, `element.remove`, `document.createElement`, `element.append`, samt ändrat egenskapen `textContent`.

>[!success] Användargränssnitt
Slutligen ska eleverna ha viss förståelse för vad användargränssnitt är och vad som behövs för att skapa bra sådana.


# Begrepp

| Begrepp                 | Betydelse                                                                                                                        |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| **HTML**                | Hyper Text Markup Language - strukturerat sätt att visa innehåll på en hemsida.                                                  |
| **DOM**                 | Document Object Model - webbläsaren läser in HTML och lägger in varje element som ett objekt som kan manipuleras med javascript. |
| **Gränssnitt/UI**       | User Interface, användargränssnitt. Sättet som användaren använder (interagerar) med applikationen/skriptet.                     |
| **Element**                        | Ett element är det som skapas av att använda en tagg i HTML (tex )                                                                                                                                 |
| **Objekt**              | Ett objekt i programmering är en grupper av egenskaper som delar samma syfte.                                                                                                                                 |
| **Egenskap (property)** | En egenskap är som en variabel men som existerar för att definiera något hos ett objekt. Håller alltså ett värde.                                                                                                                                 |
| **Metod (method)**      | En metod är en funktion som existerar på ett objekt. Används oftast för att förändra objektets egenskaper eller göra något med objektet.                                                                                                                                 |

# Agenda

## 1. Introduktion
Presenterar ämnet och vad vi ska göra. Introducerar begreppen som kommer användas, och hur lektionen kommer gå till.

1. Introduktion till hur HTML fungerar och hur det relaterar till användargränssnitt.
2. Code along där vi skapar lite HTML och testar att förändra det med javascript.
3. Avslutande övning i att använda javascript för att skicka information till ett gränssnitt (skapa element på en webbsida).

## 2. Introduktion till HTML och gränssnitt (presentation av information)
Absolut först går vi igenom vad ett användargränssnitt är, och jag visar dem några exempel på användargränssnitt, både bra och dåliga.

Vi kommer sedan titta på HTML och hur information presenteras på en hemsida. Genomgång av tekniken bakom och hur det funkar rent praktiskt.

Jag visar dem inspektionsverktyget i Chrome (finns även i Firefox och Edge) och hur man navigerar det. Visar både Console och Element-tabben.

Eleverna ska gå in på en nyhetssida, högerklicka och ändra texten på en rubrik.

## Rast

## 3. Code Along HTML och Javascript

Vi skapar ett HTML-dokument som vi fyller med några element (h1, p, span). Vi testar dem genom att öppna dem i webbläsaren. För detta behöver vi installera ett extension; *Open In Browser*.

Sedan lägger vi till script-taggen och testar att ändra rubriken, paragrafen, och span-taggen. För detta behöver vi introducera IDn och metoden `document.getElementByID("")` samt egenskapen `textContent`.

Sedan testar vi att ta bort paragrafen med metoden `remove()`.

Avslutningsvis använder vi metoderna `createElement("")` och `append()` för att skapa ett nytt element - en paragraf som innehåller en bit text, och som läggs till i slutet av sidan. Vi använder body för detta.

## 4. Tre avslutande övningar


### Övning 1
Lägg till en paragraf med en span i sig på din sida, så här;
```html
<p>Coin toss: <span id="coin-result"></span></p>
```

Följande kod skriver ut 0 eller 1 i konsolen. Använd och modifiera koden så att den istället ändrar spanen `coin-result` så att den antingen visar Heads eller Tails, där Heads är om result blir 1 och Tails är om resultatet blir 0.

```javascript
let result = Math.round(Math.random());
console.log(result);
```


### Övning 2

Bing AI har genererat tre skämt som du ska lägga till i varsin paragraf på din sida med hjälp av javascript:

1.  Varför var programmeraren så dålig på att simma? Han försökte använda en tangentbordsgenväg för att simma snabbare.
2. Varför var datorn kall? Den hade öppnat Windows.
3. Varför var spelaren så bra på att spela? Han hade övat i flera timmar varje dag.

Skriv koden så effektivt som du kan, med tekniker vi gått igenom i kursen.

### Övning 3
Skapa en counter som räknar sekunder (ditt skript ska alltså uppdatera textinnehållet i ett element en gång varje sekund) - detta krävs att du hittar information om hur man hanterar tidsintervall i javascript. Du kan använda internetsökning eller en AI för den hjälpen.
